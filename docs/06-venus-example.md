## En attendant

Pratiquez sur vos propres cas :

1. **Identifiez une regle metier** dans votre code
2. **Isolez-la** dans le Domain (pas de framework)
3. **Ecrivez le premier test** qui echoue
4. **Iterez** avec le cycle Red-Green-Refactor

## Bons candidats pour le TDD

- Calculs de prix, remises, taxes
- Validation de donnees metier
- Regles de workflow (etats, transitions)
- Algorithmes de tri, filtrage
- Transformations de donnees

## Mauvais candidats pour le TDD

- CRUD simple sans logique
- Configuration d'infrastructure
- UI/Templates
- Code de migration
